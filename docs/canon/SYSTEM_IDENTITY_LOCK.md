# SYSTEM_IDENTITY_LOCK

## Canonical workspace names
Preserve these names exactly:
- `SANDBOX`
- `XPS_AUTO_BUILDER`
- `XPS_VISION_CORTEX`
- `XPS_INTEL_CORE`
- `XPS_ADMIN_CONTROL_PLANE`
- `XPS_DISCOVERY_SYSTEM`

## Identity law
`SANDBOX` is the controlled experiment workspace.
`XPS_AUTO_BUILDER` is the bounded universal builder.
A target system may be built from an idea, but the target system does not replace these canonical names.

## Target-system rule
Every built system must be represented by:
- target profile name
- mission
- delivery level
- source-of-truth order
- write surfaces
- release criteria

## Forbidden actions
- renaming canonical XPS systems during production
- silently merging systems into one label
- treating a target package as the builder identity
- redesigning locked architecture without explicit approval
